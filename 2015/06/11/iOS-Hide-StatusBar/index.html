iOS 隱藏／顯示 Status Bar

發表於 2015-06-11   |  

首先說明 info.plist 的設定

  1. 若 View controller-based status bar appearance = YES, 則 UIViewController 對 status bar 的設定優先於 UIApplication, 並搭配UIViewController 的 3 個 API

    對 Status Bar 做動畫的更新.

  2. 若 View controller-based status bar appearance = NO, 則 UIApplication 對 status bar 的設定優先於 UIViewController 且 prefersStatusBarHidden 不會被執行，若要隱藏／顯示 Status Bar 則要使用 **UIApplication 的 2 個 API

由於 UIApplication.setStatusBarHidden withAnimation 跟 UIApplication.setStatusBarHidden 在 iOS 9 被癈除了，所以這裡就只介紹基於 iOS 8.0 之後的版本並且 info.plist 裡的 View controller-based status bar appearance = YES的做法。

執行步驟如下：
1.首先 prefersStatusBarHidden() 回傳的布林值會決定 隱藏／顯示 Status Bar
2.接著設定 preferredStatusBarUpdateAnimation() 回傳的值來決定更新 Status Bar 的動畫是那一種
3.最後呼叫 setNeedsStatusBarAppearanceUpdate(), 這個 API 會執行以下的動作

  1. 根據 prefersStatusBarHidden() 回傳的值來決定要不要執行更新 Status Bar 的狀態
  2. 若 prefersStatusBarHidden() 回傳的值表示要更新 Status Bar ，那再根據 preferredStatusBarUpdateAnimation() 回傳的值來決定更新 Status Bar 的動畫是那一種

完整的範例如下：

  1. 每按一次按鈕， isHidden 的值的都會被改變，並呼叫 UIViewController.setNeedsStatusBarAppearanceUpdate()來執行是否需要更新 Status Bar.
  2. prefersStatusBarHidden() 回傳 isHidden, 如果 isHidden = True 更新 Status Bar ,反之則否.
  3. preferredStatusBarUpdateAnimation() 回傳 UIStatusBarAnimation.Slide(Status Bar 向上滑出螢幕).

UIStatusBarAnimation 所宣告的值

UIStatusBarAnimation:

UIStatusBarAnimation 說明

UIStatusBarAnimationNone (None)
No animation is applied to the status bar as it is shown or hidden.
Available in iOS 3.2 and later.

UIStatusBarAnimationFade (Fade)
The status bar fades in and out as it is shown or hidden, respectively.
Available in iOS 3.2 and later.

UIStatusBarAnimationSlide (Slide)
The status bar slides in or out as it is shown or hidden, respectively.
Available in iOS 3.2 and later.

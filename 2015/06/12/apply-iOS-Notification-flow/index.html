iOS 申請 推播 (Push Notification) 流程

iOS 申請 推播 (Push Notification) 流程

程式碼:https://gist.github.com/sawapi/a7cee65e4ad95578044d

1.鑰匙圈存取->憑證輔助程式->從憑證授權要求憑證 =>會產生CertificateSigningRequest.certSigningRequest(憑證申請書)
2.將上步驟獲取之檔案到apple developer 申請憑證 aps_development.cer
3.將上步驟申請之憑證,透過鑰匙圈存取 轉檔成private_key.p12(按右鍵輸出)
4.將aps_development.cer轉換為 aps_development.pem
openssl x509 -in aps_development.cer -inform DER -out aps_development.pem -outform PEM
5.將private_key.p12 轉換為 private_key.pem
openssl pkcs12 -nocerts -out private_key.pem -in private_key.p12
6.去除轉發密碼
openssl rsa -out private_key_noenc.pem -in private_key.pem
7.合併 aps_development.pem 與 private_key_noenc.pem
cat aps_development.pem private_key_noenc.pem > apns_dev.pem
8.金鑰申請完成

到步驟 8 所產生的金鑰是給伺服器使用的

9.到apple developer 申請 Provisioning Profiles會產生xxxxx.mobileprovision檔,點擊後,重新run app就會產生push token


步驟 9 所產生的 xxxxx.mobileprovision檔 是給 APP用的，抓下來後在 Xcode 己開啟的情況下雙點擊xxxxx.mobileprovision檔案，Xcode 會自動載入到相同的 Bundle Identifier 的專案

10 Case： 如果建立新的專案，名稱與舊的專案不同，但想將舊的更新掉，可以如下的做法
10-1. 在 project -> targety -> info -> Bundle Identifier : 改成舊的專案 Bundle Identifier （Apple 只認 Bundle Identifier）
10-2. 如果舊的專案已經有使用 Push Notification，想要在新的專案繼續使用
10-2-1. 建議重做 1 ～9 步驟，但因為原本舊的專案已經產生過憑證（.cer），所以直接從步驟2. 做到 iOS APP -> Identifiers -> APP ID -> 找舊的專案點選 -> Edit -> Development SSL Certificate -> download 接著從步驟 2 開始
10-2-2. Provision Profiles -> All -> 找舊的專案點選 -> Download -> 下載下來後點擊會自動加入Xcode （跟步驟 9 一樣，會認 Bundle Identifier）

如何控制 iOS App 裡 View 的轉向 (How to control View's Orientation)

基本上 控制 View 的轉向主要有三個 Method 可以呼叫

override func supportedInterfaceOrientations() -> UIInterfaceOrientationMask
override func shouldAutorotate() -> Bool
override func preferredInterfaceOrientationForPresentation() -> UIInterfaceOrientation

不過我只使用其中兩種即可控制 View 的轉向，以下是範例：
在你想要View 是顯示在 Portrait 的 View Controller 裡加上以下兩個 method:

override func supportedInterfaceOrientations() -> UIInterfaceOrientationMask {
return [UIInterfaceOrientationMask.Portrait ,UIInterfaceOrientationMask.PortraitUpsideDown]
}
override func shouldAutorotate() -> Bool {
if (UIDevice.currentDevice().orientation == UIDeviceOrientation.LandscapeLeft ||
UIDevice.currentDevice().orientation == UIDeviceOrientation.LandscapeRight ||
UIDevice.currentDevice().orientation == UIDeviceOrientation.Unknown) {
return false
}
else {
return true
}
}

如果你是使用在 root View Controller 是 UINavigationController 的情況下，因為當發生 Rotation 時，UINavigationController 的 shouldAutorotate()會被執行，而它不會將此訊息往它的 child View Controller 傳，所以你實作在 ViewController 裡的 shouldAutorotate() 就不會被執行，所以你要幫 UINavigationController 加上 Extension 如下：

extension UINavigationController {

public override func shouldAutorotate() -> Bool {
    return (self.topViewController?.shouldAutorotate())!
}

}

如此，UINavigationController就會去執行目前顯示在畫面的 ViewController 的 shouldAutorotate()

That’s it

iOS 動畫： 模擬文字向上／向下翻轉的動畫

這篇是從這位 作者 的 github 節錄下來的，若對 iOS 的動畫有興趣，可以關注這位作者

直接上 Code


import UIKit
import QuartzCore

enum AnimationDirection: Int {
    case Positive = 1
    case Negative = -1
}
class thirdViewController: UIViewController {

    @IBOutlet weak var bgImage: UIImageView!
    @IBOutlet weak var label1: UILabel!
    override func viewDidLoad() {
        super.viewDidLoad()
        self.repeat1()
        // Do any additional setup after loading the view.
    }
    func repeat1(){
        delay(seconds: 1){
            self.cubeTransition(label: self.label1, text: "SuperXlx", direction: AnimationDirection.Positive)
            self.fadeImageView(self.bgImage, toImage: UIImage(named: "bg-sunny")!)
            self.repeat2()
        }
    }
    func repeat2(){
        delay(seconds: 1){
            self.fadeImageView(self.bgImage, toImage: UIImage(named: "bg-snowy")!)
            self.cubeTransition(label: self.label1, text: "StrongX", direction: AnimationDirection.Negative)
            self.repeat1()
        }
    }
    override func didReceiveMemoryWarning() {
        super.didReceiveMemoryWarning()
        // Dispose of any resources that can be recreated.
    }
    func fadeImageView(imageView: UIImageView, toImage: UIImage) {

        UIView.transitionWithView(imageView, duration: 1.0,
            options: .TransitionCrossDissolve, animations: {
                imageView.image = toImage
            }, completion: nil)

    }

    func cubeTransition(#label: UILabel, text: String, direction: AnimationDirection) {

        let auxLabel = UILabel(frame: label.frame)
        auxLabel.text = text
        auxLabel.font = label.font
        auxLabel.textAlignment = label.textAlignment
        auxLabel.textColor = label.textColor
      //  auxLabel.backgroundColor = label.backgroundColor

        // A1,A2: 先將 auxLabel 顯示成高度 y = 0.1 的狀態，也就是壓扁的樣子
        // A1.
        let auxLabelOffset = CGFloat(direction.rawValue) * label.frame.size.height/2.0
        // A2.
        auxLabel.transform = CGAffineTransformConcat(
            CGAffineTransformMakeScale(1.0, 0.1),
            CGAffineTransformMakeTranslation(0.0, auxLabelOffset))

        // A3. 將 auxLabel 加入 subView
        label.superview!.addSubview(auxLabel)

        // A4. 執行動面
        UIView.animateWithDuration(0.5, delay: 0.0, options: .CurveEaseOut, animations: {
            // A4-1. 將 auxLabel 回復成跟原本的 label 一樣高度
            auxLabel.transform = CGAffineTransformIdentity


            // A4-2. 將原本的 label 的高度壓成 y = 0.1 的大小，並向上移動，達到向上翻轉的效果
            label.transform = CGAffineTransformConcat(
                // A4-2-1. label 高度縮小
                CGAffineTransformMakeScale(1.0, 0.1),
                // A4-2-2. label 向上移動
                CGAffineTransformMakeTranslation(0.0, -auxLabelOffset))
            }, completion: {_ in

                // 最後還是由 label 顯示最後的文字，並把 auxLabel 移除
                label.text = auxLabel.text
                label.transform = CGAffineTransformIdentity

                auxLabel.removeFromSuperview()
        })

    }
}

iOS 設定 UIView 的某幾個角為圓角

OK, I just copy from someone kerker~~

Objective-C 版本

UIView *view2 = [[UIView alloc] initWithFrame:CGRectMake(120, 10, 80, 80)];
view2.backgroundColor = [UIColor redColor];
[self.view addSubview:view2];

UIBezierPath maskPath = [UIBezierPath bezierPathWithRoundedRect:view2.bounds byRoundingCorners:UIRectCornerBottomLeft | UIRectCornerBottomRight cornerRadii:CGSizeMake(10, 10)];
CAShapeLayer maskLayer = [[CAShapeLayer alloc] init];
maskLayer.frame = view2.bounds;
maskLayer.path = maskPath.CGPath;
view2.layer.mask = maskLayer;

Swift 版本
// 1. Need to be Rounded View
let roundedView = UIView(frame: CGRect(x: 0, y: 0, width: 120, height: 120))
roundedView.backgroundColor = UIColor.redColor()

// 2. Setup BezierPath for rounding corners
let maskPath = UIBezierPath(roundedRect: roundedView.bounds, byRoundingCorners: UIRectCorner.BottomLeft | UIRectCorner.BottomRight, cornerRadii: CGSize(width: 5, height: 5))

// 3. Instant and configure a CAShaperLayer for mask view
let maskLayer = CAShaperLayer()
maskLayer.frame = roundedView.bounds
maskLayer.path = maskPath.CGPath

// 4. maske roundedView by “maskLayer”
roundedView.layer.mask = maskLayer

客製化 UITextView 鍵盤的 inputAccessoryView

在使用 UITextView 的時候，我們可以對鍵盤的 inputAccessoryView 做簡單的客製化

inputAccessoryView 就是鍵盤最上面的那條 Tool Bar 如下圖所示

程式碼如下

// 建立一個 UIToolBar


let keyboardToolBar = UIToolbar()
keyboardToolBar.sizeToFit()

// 建立所要的 Bar Button


let done = UIBarButtonItem(barButtonSystemItem: UIBarButtonSystemItem.Done, target: self, action: "endEditingNow")

let takePicture = UIBarButtonItem(barButtonSystemItem: UIBarButtonSystemItem.Camera, target: self, action: "openCameraOrAlbum")

let flexible = UIBarButtonItem(barButtonSystemItem: UIBarButtonSystemItem.FlexibleSpace, target: nil, action: nil)

let fixedSpace = UIBarButtonItem(barButtonSystemItem: UIBarButtonSystemItem.FixedSpace, target: nil, action: nil)

fixedSpace.width = 20.0

// 將所有的 Bar Button 加入 Tool Bar


var toolbarButtons = [flexible, takePicture, fixedSpace, done, flexible]
keyboardToolBar.setItems(toolbarButtons, animated: false)

// 將TextView(UITextView)的 inputAccessoryView 設成此 Tool Bar 


textView.inputAccessoryView = keyboardToolBar

上述的程式碼可以加在 viewDidLoad 裡

目前我測試的狀況是，用 UIToolBar 的話這個 Tool Bar 可以很完美的隨著鍵盤上升下降在正確的位置，如果是用 UIView 的話位置會不正常

或許用UIView的話也是可行的，只是再多做一些設定，不過以我目前的狀況這樣就夠用了，等之後遇到再說吧

iOS 實作 圖文編輯器 （二） UITextView 設定

##根據 iOS 實作 圖文編輯器（一）裡的 UITextViewDelegate

1. 我們可以設定當游標移到那，UITextView 就會將游標所在的範圍移到可視範圍


func textViewDidChangeSelection(textView: UITextView) {        
  textView.scrollRangeToVisible(textView.selectedRange)
}

2. 當鍵盤上升時，會自動的將游標那行上移到可視範圍


func textViewDidChange(textView: UITextView) {

  // 游標的位置
  let line = textView.caretRectForPosition(textView.selectedTextRange!.start)

  // 搭配 UIKeyboardWillShowNotification 時將鍵盤高度設定給 textView.contentInset.bottom
  let overflow = line.origin.y + line.size.height - (textView.contentOffset.y + textView.bounds.size.height - textView.contentInset.bottom - textView.contentInset.top)

  if overflow > 0 {
      // We are at the bottom of the visible text and introduced
      // a line feed, scroll down (iOS 7 does not do it)
      // Scroll caret to visible area
      var offset = textView.contentOffset
      offset.y += overflow + 7; // leave 7 pixels margin

      // setContentOffset:animated: 時 animated 要設為  false 不然 caret 會消失
      UIView.animateWithDuration(1, animations: { () -> Void in
          textView.setContentOffset(offset, animated: false)
      })
  }

}

3. 鍵盤的設定

在viewDidLoad 裡設定接收UIKeyboardNotification 的方法

鍵盤由下方彈出時

NSNotificationCenter.defaultCenter().addObserver(self, selector: “keyboardWillShow:”, name: UIKeyboardWillShowNotification, object: nil)

鍵盤由收回至下方時

NSNotificationCenter.defaultCenter().addObserver(self, selector: “keyboardWillHide:”, name: UIKeyboardWillHideNotification, object: nil)


func keyboardWillShow(n:NSNotification) {
  self.keyboardShowing = true

  let d = n.userInfo!
  var r = (d[UIKeyboardFrameEndUserInfoKey] as! NSValue).CGRectValue()
  r = textView.convertRect(r, fromView:nil)
  textView.contentInset.bottom = r.size.height
  textView.scrollIndicatorInsets.bottom = r.size.height
}

func keyboardWillHide(n:NSNotification) {
  self.keyboardShowing = false
  textView.contentInset = UIEdgeInsetsZero
  textView.scrollIndicatorInsets = UIEdgeInsetsZero
}

iOS 實作 圖文編輯器 （一） UITextView 與 Text Kit


UITextView 是 UIScrollView 的子類別，基本上它跟 UITextField很像，差別在於，UITextView可以寫入多行並且可以使用 TextKit 達到圖文並茂的文章

底下列出 UITextView 的一些屬性, 代理，通知

Attribute
1. selectedRange : 目前選擇的範圍，selectedRange.location 就是游標所在的位置
2. scrollRangeToVisible : 滾動到指到的範圍

### UITextViewDelegate Protocol:
1. textViewShouldBeginEditing:
2. textViewDidBeginEdition:
3. textViewShouldEndEditing:
4. textViewDidEndEditing:
5. textView: ShouldChangeTextInRange: replacementText:
6. textView: ShouldInteractWithTextAttachment: inRange:
7. textView: shouldInteractWithURL: inRange:

### Notification
1. UITextViewTextDidBeginEditingNotification
2. UITextViewTextDidEndEditingNotification
3. UITextViewtextDidChangeNotification
4. UITextViewDidChangeSelection

之後再根據有使用到的 Method 再加入說明


Text Kit 是用來處理文字的引擎，可以將 NSAttributedString 繪製到畫面上

這邊主要討論 UITextView 跟 Text Kit 之間的關係
Text Kit 主要由3個類別組成，分別是
NSTextStorage,
NSTextStorageNSTextContainer,
NSTextStorageNSLayoutManager

而 UITextView 提供了可以直接存取 Text Kit 這 3 個類別的屬性

textStorage (NSTextStorage) : NSMutableAttributedString 的子類別，它就是儲存 UITextView裡所有的文字。

layoutManager (NSLayoutManager) : 是掌管如何將文字繪製到 textContainer 的類別，將 textStorage 裡的文字根據設定好的 layout(布局) 繪製到 textContainer

textContainer (NSTextContainer) : 供文字繪製的容器

textContainerInset :
在大部份的情況下
一個 textStorage 會有一個 layoutManager
而一個 layoutManager 會有一個 textContainer

將內建的相機介面改成中文

當使用iOS內建的照相機介面時，例如使用 UIImagePickerController 來開啟相機時預設介面會是英文的，這時只要在專案裡
點選 PROJECT -> Info -> Localizations
基本上在 Localizations 你只會看到 Engilsh - Development Language
按下 ＋ 新增 Chinese (Traditional) 即可

接著重新build一次，再開啟 app 裡的相機，這時你會發覺 cancel, retake use Photo 都變成中文囉
因為此時 iOS 就會根據裝置的語系來顯示選項
同理，之後的文字選單的 複製，貼上，全選等等都會以中文顯

the_Rule_Of_version_Number_Of_Node.JS

關於package.json裡設定 node.js 的 module 版號，使用的規則如下

假設 package.json 裡的內容如下


{
  "name": "MyApp",
  "version": "1.0.2",
  "private": true,
  "scripts": {
    "start": "node ./bin/www"
  },
  "dependencies": {
    "body-parser": "~1.12.4",
    "cookie-parser": "~1.3.5",
    "debug": "~2.2.0",
    "express": "~4.12.4",
    "jade": "~1.9.2",
    "morgan": "~1.5.3",
    "serve-favicon": "~2.2.1",
    "mongodb": "2.0.33",
    "monk": "1.0.1",
    "connect-flash": "0.1.1",
    "express-validator": "2.10.0",
    "express-session": "1.11.2",
    "express-messages": "0.0.2",
    "multer": "0.1.8",
    "moment": "2.10.3"
  }
}

以 morgan: “~1.5.3” 為例

1 : main version， 主要版號
5 : sub version，次版號，有新功能時會增加
3 : bug version, bug 版號，有修正 bug 時會增加

至於版號前面的符號所代表的意思如下：
“~1.5.3” 代表 1.5.x （Bug修正等等更新，舉例：會更新1.5.4， 1.6.1就不會）
“^1.5.3” 則相當於會更新到1.x.x （新版，有新功能，但會相容，舉例：會更新1.6.3， 2.0.1就不會）
“1.5.3” 永遠只會用1.5.3這版本（也就是如果新版修正了1.5.3的 bug， 你用 npm update 也不會更新）

基本上當你使用 npm install morgan –save 的時候
它預設會寫入 “morgan”: “＾1.5.3”

iOS 申請 推播 (Push Notification) 流程

iOS 申請 推播 (Push Notification) 流程

程式碼:https://gist.github.com/sawapi/a7cee65e4ad95578044d

1.鑰匙圈存取->憑證輔助程式->從憑證授權要求憑證 =>會產生CertificateSigningRequest.certSigningRequest(憑證申請書)
2.將上步驟獲取之檔案到apple developer 申請憑證 aps_development.cer
3.將上步驟申請之憑證,透過鑰匙圈存取 轉檔成private_key.p12(按右鍵輸出)
4.將aps_development.cer轉換為 aps_development.pem
openssl x509 -in aps_development.cer -inform DER -out aps_development.pem -outform PEM
5.將private_key.p12 轉換為 private_key.pem
openssl pkcs12 -nocerts -out private_key.pem -in private_key.p12
6.去除轉發密碼
openssl rsa -out private_key_noenc.pem -in private_key.pem
7.合併 aps_development.pem 與 private_key_noenc.pem
cat aps_development.pem private_key_noenc.pem > apns_dev.pem
8.金鑰申請完成

到步驟 8 所產生的金鑰是給伺服器使用的

9.到apple developer 申請 Provisioning Profiles會產生xxxxx.mobileprovision檔,點擊後,重新run app就會產生push token


步驟 9 所產生的 xxxxx.mobileprovision檔 是給 APP用的，抓下來後在 Xcode 己開啟的情況下雙點擊xxxxx.mobileprovision檔案，Xcode 會自動載入到相同的 Bundle Identifier 的專案

10 Case： 如果建立新的專案，名稱與舊的專案不同，但想將舊的更新掉，可以如下的做法
10-1. 在 project -> targety -> info -> Bundle Identifier : 改成舊的專案 Bundle Identifier （Apple 只認 Bundle Identifier）
10-2. 如果舊的專案已經有使用 Push Notification，想要在新的專案繼續使用
10-2-1. 建議重做 1 ～9 步驟，但因為原本舊的專案已經產生過憑證（.cer），所以直接從步驟2. 做到 iOS APP -> Identifiers -> APP ID -> 找舊的專案點選 -> Edit -> Development SSL Certificate -> download 接著從步驟 2 開始
10-2-2. Provision Profiles -> All -> 找舊的專案點選 -> Download -> 下載下來後點擊會自動加入Xcode （跟步驟 9 一樣，會認 Bundle Identifier）

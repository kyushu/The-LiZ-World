客製化 UITextView 鍵盤的 inputAccessoryView

在使用 UITextView 的時候，我們可以對鍵盤的 inputAccessoryView 做簡單的客製化

inputAccessoryView 就是鍵盤最上面的那條 Tool Bar 如下圖所示

程式碼如下

// 建立一個 UIToolBar


let keyboardToolBar = UIToolbar()
keyboardToolBar.sizeToFit()

// 建立所要的 Bar Button


let done = UIBarButtonItem(barButtonSystemItem: UIBarButtonSystemItem.Done, target: self, action: "endEditingNow")

let takePicture = UIBarButtonItem(barButtonSystemItem: UIBarButtonSystemItem.Camera, target: self, action: "openCameraOrAlbum")

let flexible = UIBarButtonItem(barButtonSystemItem: UIBarButtonSystemItem.FlexibleSpace, target: nil, action: nil)

let fixedSpace = UIBarButtonItem(barButtonSystemItem: UIBarButtonSystemItem.FixedSpace, target: nil, action: nil)

fixedSpace.width = 20.0

// 將所有的 Bar Button 加入 Tool Bar


var toolbarButtons = [flexible, takePicture, fixedSpace, done, flexible]
keyboardToolBar.setItems(toolbarButtons, animated: false)

// 將TextView(UITextView)的 inputAccessoryView 設成此 Tool Bar 


textView.inputAccessoryView = keyboardToolBar

上述的程式碼可以加在 viewDidLoad 裡

目前我測試的狀況是，用 UIToolBar 的話這個 Tool Bar 可以很完美的隨著鍵盤上升下降在正確的位置，如果是用 UIView 的話位置會不正常

或許用UIView的話也是可行的，只是再多做一些設定，不過以我目前的狀況這樣就夠用了，等之後遇到再說吧

iOS 實作 圖文編輯器 （二） UITextView 設定

##根據 iOS 實作 圖文編輯器（一）裡的 UITextViewDelegate

1. 我們可以設定當游標移到那，UITextView 就會將游標所在的範圍移到可視範圍


func textViewDidChangeSelection(textView: UITextView) {        
  textView.scrollRangeToVisible(textView.selectedRange)
}

2. 當鍵盤上升時，會自動的將游標那行上移到可視範圍


func textViewDidChange(textView: UITextView) {

  // 游標的位置
  let line = textView.caretRectForPosition(textView.selectedTextRange!.start)

  // 搭配 UIKeyboardWillShowNotification 時將鍵盤高度設定給 textView.contentInset.bottom
  let overflow = line.origin.y + line.size.height - (textView.contentOffset.y + textView.bounds.size.height - textView.contentInset.bottom - textView.contentInset.top)

  if overflow > 0 {
      // We are at the bottom of the visible text and introduced
      // a line feed, scroll down (iOS 7 does not do it)
      // Scroll caret to visible area
      var offset = textView.contentOffset
      offset.y += overflow + 7; // leave 7 pixels margin

      // setContentOffset:animated: 時 animated 要設為  false 不然 caret 會消失
      UIView.animateWithDuration(1, animations: { () -> Void in
          textView.setContentOffset(offset, animated: false)
      })
  }

}

3. 鍵盤的設定

在viewDidLoad 裡設定接收UIKeyboardNotification 的方法

鍵盤由下方彈出時

NSNotificationCenter.defaultCenter().addObserver(self, selector: “keyboardWillShow:”, name: UIKeyboardWillShowNotification, object: nil)

鍵盤由收回至下方時

NSNotificationCenter.defaultCenter().addObserver(self, selector: “keyboardWillHide:”, name: UIKeyboardWillHideNotification, object: nil)


func keyboardWillShow(n:NSNotification) {
  self.keyboardShowing = true

  let d = n.userInfo!
  var r = (d[UIKeyboardFrameEndUserInfoKey] as! NSValue).CGRectValue()
  r = textView.convertRect(r, fromView:nil)
  textView.contentInset.bottom = r.size.height
  textView.scrollIndicatorInsets.bottom = r.size.height
}

func keyboardWillHide(n:NSNotification) {
  self.keyboardShowing = false
  textView.contentInset = UIEdgeInsetsZero
  textView.scrollIndicatorInsets = UIEdgeInsetsZero
}

iOS 實作 圖文編輯器 （一） UITextView 與 Text Kit

UITextView 是 UIScrollView 的子類別，基本上它跟 UITextField很像，差別在於，UITextView可以寫入多行並且可以使用 TextKit 達到圖文並茂的文章

底下列出 UITextView 的一些屬性, 代理，通知

Attribute
1. selectedRange : 目前選擇的範圍，selectedRange.location 就是游標所在的位置
2. scrollRangeToVisible : 滾動到指到的範圍

### UITextViewDelegate Protocol:
1. textViewShouldBeginEditing:
2. textViewDidBeginEdition:
3. textViewShouldEndEditing:
4. textViewDidEndEditing:
5. textView: ShouldChangeTextInRange: replacementText:
6. textView: ShouldInteractWithTextAttachment: inRange:
7. textView: shouldInteractWithURL: inRange:

### Notification
1. UITextViewTextDidBeginEditingNotification
2. UITextViewTextDidEndEditingNotification
3. UITextViewtextDidChangeNotification
4. UITextViewDidChangeSelection

之後再根據有使用到的 Method 再加入說明


Text Kit 是用來處理文字的引擎，可以將 NSAttributedString 繪製到畫面上

這邊主要討論 UITextView 跟 Text Kit 之間的關係
Text Kit 主要由3個類別組成，分別是
NSTextStorage,
NSTextStorageNSTextContainer,
NSTextStorageNSLayoutManager

而 UITextView 提供了可以直接存取 Text Kit 這 3 個類別的屬性

textStorage (NSTextStorage) : NSMutableAttributedString 的子類別，它就是儲存 UITextView裡所有的文字。

layoutManager (NSLayoutManager) : 是掌管如何將文字繪製到 textContainer 的類別，將 textStorage 裡的文字根據設定好的 layout(布局) 繪製到 textContainer

textContainer (NSTextContainer) : 供文字繪製的容器

textContainerInset :
在大部份的情況下
一個 textStorage 會有一個 layoutManager
而一個 layoutManager 會有一個 textContainer
